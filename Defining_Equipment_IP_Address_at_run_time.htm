18.7.3 Définition de l'adresse IP de l'équipement lors du runtime

Les exemples suivants montrent comment lire et écrire l'adresse IP d'un équipement lors du runtime à l'aide des méthodes d'équipement décrites à la section 18.3.8.10, Méthodes d'équipement.

 

Valeur de lecture

// --------------- Lecture des données existantes --------------------------------------

// données[0] vers données[3], conserver l'adresse IP.

// data.length peut être utilisé pour interroger le nombre d'éléments dans le tableau.

int [] data = ModbusEquipment01.getProperty( "Con_IPAddress" );

// Utiliser les données reçues récemment.

for( int i = 0;  i < data.length;  i++ )

{

    PanelIPData.IP[i].write( data[i] );

}

 

Valeur d'écriture

// --------------- Écrire des données -----------------------------------------

// La longueur est donnée par NewIP_InputData.length

int [] NewIP_InputData = {10, 187, 225, 135};

ModbusEquipment01.openConfig();

ModbusEquipment01.setProperty( "Con_IPAddress", NewIP_InputData );

ModbusEquipment01.commitConfig();

// Ce code permet de définir l'adresse IP de n'importe quel pilote, non seulement le pilote Modbus.
