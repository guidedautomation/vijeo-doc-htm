flushToStorage

Fonction

Lorsqu'une variable est associée à un groupe de journalisation de données, vous pouvez utiliser flushToStorage pour déplacer les données de la variable à partir du buffer vers un support de stockage permanent.

La taille du buffer est configurable. Vous pouvez configurer la taille du buffer dans la propriété Nbre de registres dans la RAM dans l'inspecteur de propriétés du groupe de journalisation. Lorsqu'il est saturé, les enregistrements des variables sont déplacés vers un support de stockage afin que de nouveaux enregistrements puissent être ajoutés au buffer.

Vous souhaiterez peut-être exécuter flushToStorage :

  • à des intervalles réguliers (script d'application Périodique) ;

  • à chaque modification d'écran (script d'écran A la fermeture) ;

  • lorsque vous effectuez la maintenance

  • lorsque vous modifiez un emplacement de stockage (carte CF/SD ou lecteur USB)

  • Cette méthode transfert les données d'une variable unique vers le stockage. Cette méthode ne transfert pas le buffer au complet contenant toutes les données historiques de toutes les variables vers le stockage.

  • Lorsque vous dépendez d'un vidage automatique des données du buffer vers le support de stockage, sachez que toutes les données ne sont pas forcément supprimées du buffer. Utilisez flushToStorage pour assurer le déplacement de toutes les données du buffer vers le support de stockage.

  • Les données figurant dans la SRAM ne peuvent être vidées qu'une seule fois. Si vous utilisez la fonction Copier sur USB et choisissez de supprimer les fichiers de la source après la copie, aucun nouveau fichier n'est créé sauf s'il existe de nouvelles données dans la SRAM.

Format

boolean VariableName.flushToStorage();

Valeur retournée

Retourne la valeur true lorsque les données historiques dans le buffer sont déplacées avec succès dans le fichier électronique dans le lecteur secondaire ou optionnel.

Renvoie la valeur false lorsque la variable ne stocke pas son historique. En d'autres mots, la variable n'est pas associée au groupe de journalisation de données.
