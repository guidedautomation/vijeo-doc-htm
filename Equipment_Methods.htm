18.9.1.10 Méthodes d'équipement

Utilisez les méthodes suivantes afin d'interagir avec l'équipement.

Méthode

Description

commitConfig

Permet d'enregistrer l'adresse IP modifiée dans la SRAM et de réinitialiser la connexion Ethernet. Vous ne devez pas redémarrer la machine cible pour appliquer les modifications.

getProperty

Retourne un tableau entier contenant des valeurs de propriété d'équipement.

getStringProperty

Retourne une chaîne unique contenant le nom du pilote de l'équipement ou l'adresse de départ de la table de dialogue.

openConfig

Ouvre les paramètres de configuration de l'équipement.

setProperty

Permet d'écrire le paramètre de propriété spécifié dans la SRAM de l'équipement.

 

Exemple

Les exemples suivants illustrent comment configurer des adresses esclaves et une adresse IP d'équipement à l'aide des méthodes d'équipement. Pour consulter plus d'exemples, reportez-vous à la section 18.7.3, Définition de l'adresse IP de l'équipement lors de l'exécution.

 

Configuration d'un équipement Modbus RTU

int [] slaveAddress = {255};

ModbusEquipment01.openConfig();

ModbusEquipment01.setProperty( "Con_SlaveEquipmentAddress", slaveAddress );

ModbusEquipment01.commitConfig();

 

Configuration d'un équipement esclave Modbus

int [] slaveAddress = {255};

ModbusSlaveRTU01.openConfig();

ModbusSlaveRTU01.setProperty("Con_EquipmentAddress", slaveAddress);

ModbusSlaveRTU01.commitConfig();

 

Configuration d'un équipement Modbus TCP

int [] ipAddress = {192, 168, 1, 10};

ModbusEquipment01.openConfig();

ModbusEquipment01.setProperty( "Con_IPAddress", ipAddress );

ModbusEquipment01.commitConfig();
