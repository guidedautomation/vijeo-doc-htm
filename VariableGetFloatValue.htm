getFloatValue

Fonction

Permet au script d'interagir avec les variables Vijeo-Designer. Les variables du type BOOL, entier et REAL peuvent retourner une valeur flottante.

  • Avant d'utiliser les variables partagées ou externes, vous devriez vérifier l'intégrité des données (isDataQualityGood).

Format

float NomVariable.getFloatValue();

float NomVariableBloc.getFloatValue(int index);

Paramètre

Paramètre

Description

int index

Pour les variables de bloc, spécifiez la position d'index de l'élément duquel vous obtenez les données. Les numéros d'élément débutent à 0.

  • Après avoir utilisé des valeurs de variable de bloc, utilisez la méthode write pour modifier le bloc sur la machine cible, puis utilisez la méthode sendData pour écrire les valeurs vers l'équipement.

Valeur retournée

Retourne la valeur d'une variable Vijeo-Designer de type REAL.

Retourne 0 lorsque vous accédez à un élément de variable de bloc qui est hors limites.

  • Il n'existe aucune façon pour vérifier une bonne valeur de 0 ou une mauvaise valeur (index de tableau hors limites) de 0. L'écran affiche un message d'événement Runtime pour indiquer un accès invalide à la variable de bloc, mais il n'existe aucune façon pour automatiser cette vérification.

  • Lorsque vous utilisez des scripts pour obtenir la valeur d'un élément spécifique dans une variable de bloc, et que l'index que vous spécifiez n'existe pas, le script retournera tout simplement la valeur «0». Vérifiez si les accès d'index dans les scripts vers une variable de bloc s'exécutent correctement.
