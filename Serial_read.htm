read

Fonction

Lit le flux d'octets depuis le port de communication défini.

Vous pouvez utiliser les méthodes read/write du pilote de script pour réaliser les communications de base entre la machine cible et un équipement. Cet équipement peut être une imprimante, un équipement ou une autre machine cible.

Format

int NomPiloteScript.read(

int timeout,

int numOfBytes,

byte[] data );

Paramètres

Paramètre

Description

int timeout

Spécifie la durée pendant laquelle la méthode read attend les données avant de s'arrêter. Dans la plupart des cas, timeout devrait être zéro.

  • Lorsqu'il n'y a aucune donnée à lire à partir du port série, le pilote de script attend pour la période timeout définie. Lors du timeout, le script bloque l'exécution des opérations du même type qui peuvent ignorer les saisies d'opérateur ou ralentir les scripts périodiques. Utilisez cette valeur uniquement lorsque vous avez réellement besoin d'un timeout.

  • Ne spécifiez pas un long timeout si vous attendez que des données ne soient pas disponibles pour la lecture.

  • Pour plus d'informations sur les effets secondaires lors de l'utilisation d'un timeout, reportez-vous à la section 18.8.1, Points importants concernant le pilote de script.

int numOfBytes

Définit le nombre attendu d'octets dans le flux d'octets.

  • Le montant maximum de données que vous pouvez lire est de 1024 octets. Pour lire plus de données, exécutez la méthode read de nouveau.

  • numOfBytes et le nombre d'éléments dans le tableau d'octets devraient correspondre. Si numOfBytes est plus grand, il se peut que vous lisiez plus de données pouvant être stockées.

byte[] data

Zone de stockage du flux d'octets. Si vous lisez des données à partir du port série, elles sont stockés dans ce tableau d'octets.

  • Lorsque vous référencez les données dans le tableau, n'oubliez pas que les références de tableau commencent à partir de 0.
    Donc, pour lire le huitième octet dans le flux d'octets, utilisez data[7].

Valeur retournée

Nombre d'octets reçus. La valeur (0) indique qu'une erreur est survenue.

  • La valeur retournée maximale est numOfBytes. Même si le buffer du pilote de script contient plus de données, numOfBytes le nombre maximum d'octets qui sont lus.
    Par exemple, s'il y a 110 octets de données dans le buffer du pilote de script et numOfBytes est de 100 :

  • lit 100 octets

  • valeur retournée est de 100

  • 10 octets demeurent dans le buffer du pilote de script.

S'il y a 10 octets de données dans le buffer du pilote de script et numOfBytes est de 10 :

  • lit 10 octets

  • valeur retournée est de 10

  • 0 octets demeurent dans le buffer du pilote de script.

Commentaires

Pour utiliser les méthodes read/write de pilote de script, vous devez tout d'abord configurer un pilote de script. Reportez-vous à la section 18.8.3, Configuration du pilote de script.
