18.4 Recherche des erreurs

Il est recommandé de vérifier que votre script est exempt d'erreurs au fur et à mesure que vous l'écrivez. Le débogage se complique si vous attendez la fin de la rédaction de vos scripts, notamment si ces derniers font appel à une logique complexe.

Pour vérifier que votre script est exempt d'erreurs, cliquez avec le bouton droit de la souris dans l'Editeur de scripts, puis sélectionnez Valider.

La fenêtre Compte-rendu affiche les résultats de la validation.

Cliquez deux fois sur le message d'erreur dans la fenêtre Compte-rendu. Le script s'ouvre et la flèche du pointeur se transforme en point d'insertion dans la ligne du script à l'origine de l'erreur. Dans la majorité des cas, le point d'insertion apparaît sous la ligne du script à l'origine de l'erreur.

  • L'éditeur peut uniquement trouver les problèmes liés à la syntaxe. Les problèmes liés à la logique de programmation peuvent produire des erreurs " NullPointerException " ou d'autres erreurs d'exécution Java, mais n'afficheront pas les erreurs dans la zone de rétroaction.

 

Erreurs les plus courantes dans les scripts

  • Erreur de syntaxe : indique un problème de formatage de votre script.
    Une erreur de syntaxe peut se produire lorsque l'un des éléments suivants est manquant :

  • Variable locale non déclarée : vous essayez peut-être d'utiliser une variable Vijeo-Designer ou un autre objet directement dans le script. Faites appel aux méthodes pour interagir avec les objets Vijeo-Designer.

  • Le type Void est utilisé uniquement comme type de retour de fonction.l'une des expressions a abouti à une valeur Void et ne peut pas faire l'objet d'un traitement, ce qui peut indiquer que vous avez ajouté un paramètre à une méthode qui n'en accepte pas.

  • La conversion au type "" à partir du type "" peut entraîner la perte de données.une valeur est affectée à une variable dont les types de données ne correspondent pas. Dans certains cas, cette opération est acceptable ; elle peut toutefois entraîner une perte de données dans ce cas précis. Pour poursuivre cette opération d'affectation, typez la valeur.

Exemple :

int a = 0;

double b = 0.0;

a = b;      // Échec lors de la validation

a = (int)b; // Typage pour permettre la réussite
