delete

Fonction

Le but de ce script est de retirer l'alarme à partir de la liste d'alarmes affichée par le résumé d'alarmes.

Il existe trois listes d'alarmes : Actif, Historique et Journal. Le résumé d'alarmes est associé avec l'une de ces listes. Lorsque vous exécutez la méthode delete dans le résumé d'alarmes, l'alarme est supprimée de la liste d'alarmes associée. Cette opération est différente de la méthode clear, qui supprime l'alarme des trois listes d'alarmes.

Une autre différence entre les méthodes delete et clear est que la méthode delete, après qu'une alarme est supprimée, affiche un message d'alarme lorsque l'état de l'alarme est modifié (p. ex. : Haut à Très haut, Haut à Normal, etc.).

Vous pouvez utiliser un paramètre pour vous assurer que seul le type d'alarme spécifié est supprimé.

  • Lorsque vous l'utilisez avec un groupe d'alarmes de diagnostic, cette méthode de script ne touche pas les alarmes dans le tampon de diagnostic de l'automate. Cette méthode de script n'interagit qu'avec l'affichage du résumé d'alarmes. Vous pouvez utilisez cette méthode sur tous les messages d'alarme dans les affichages d'alarme historiques et de journal, et sur les messages d'alarme acquittés dans les affichages d'alarme actifs.

Format

int NomRésuméAlarme.delete(int Msg_State);

Paramètres

Paramètre

Description

int Msg_State

Spécifie le type d'alarmes effacé dans cette opération.

0: Permet de supprimer le message d'alarme, quel que soit l'état dans lequel il se trouve.

1: Permet de supprimer un message d'alarme d'état Actif ou Non acquitté.

2: Permet de supprimer un message d'alarme d'état Retombé à la normale (Retombé).

3: Permet de supprimer un message d'alarme d'état Acquitté.

4: Permet de supprimer un message d'alarme d'état Actif.

5: Permet de supprimer un message d'alarme d'état Non acquitté.

Valeur retournée

Retourne un entier - 1 si l'alarme est supprimée, 0 si l'alarme n'est pas supprimée.

Voir également

clear, clearAll, deleteAll

  • Si vous ouvrez un projet crée sous la version 4.1 ou sous une version antérieure, toute utilisation de clear dans un script sera remplacé par delete.
