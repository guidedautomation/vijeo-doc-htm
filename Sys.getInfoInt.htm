Sys.getInfoInt

Fonction

La méthode système getInfoInt() permet de récupérer une valeur entière qui représente l'informaton système selon le paramètre passé à la méthode. getInfoInt() n'effectuera pas une scrutation en continu pour trouver des modifications aux données. Vous devez demander de nouvelles valeurs chaque fois que vous en avez besoin. La valeur est récupérée en tant qu'entier.

Format

int Sys.getInfoInt(int param);

Paramètres

Paramètre

Description

int param

Une valeur entière qui représente les informations demandées.

  • TOTAL_DRAM_USAGE
    Retourne l'utilisation total de DRAM de la cible.

  • FREE_KBYTE_SECONDARY
    Retourne en kilooctets le montant d'espace disponible dans le lecteur secondaire de la cible.

  • FREE_KBYTE_OPTIONAL
    Retourne en kilooctets le montant d'espace disponible dans le lecteur optionnel de la cible.

  • FREE_KBYTE_MAIN
    Retourne en kilooctets le montant d'espace disponible dans le lecteur principal de la cible.

  • HMI_APPLICATION_CHECKSUM
    Retourne la somme de contrôle de l'application. Ce paramètre est pris en charge uniquement lorsque la propriété Vérifier l'intégrité du projet est activée dans les propriétés de la cible. Pour plus d'informations, reportez-vous à la section 4.5.1, Propriétés de la cible.

  • STATION_NUMBER
    Retourne l'ID de la boîte de jonction connectée sur la machine cible XBTGH. Ce paramètre est pris en charge uniquement sur les cibles XBTGH connectées sur une boîte de jonction.

Exemple :

Sys.getInfoInt(TOTAL_DRAM_USAGE)

Valeur retournée

Retourne 0 ou -1 si la propriété n'est pas prise en charge par la machine cible.

Autres valeurs retournées :

  • TOTAL_DRAM_USAGE : une valeur entière comprise entre 0 et 2 147 483 648 Ko - retourne 0 si la valeur de mémoire est inférieure à 1 Ko.

  • FREE_KBYTE_SECONDARY : une valeur entière comprise entre 0 et 2 147 483 648 Ko - retourne 0 si la valeur de mémoire est inférieure à 1 Ko.

  • FREE_KBYTE_OPTIONAL : une valeur entière comprise entre 0 et 2 147 483 648 Ko - retourne 0 si la valeur de mémoire est inférieure à 1 Ko.

  • FREE_KBYTE_MAIN : une valeur entière comprise entre 0 et 2 147 483 648 Ko - retourne 0 si la valeur de mémoire est inférieure à 1 Ko.

  • HMI_APPLICATION_CHECKSUM : une valeur entière non signée 32 bits.

  • STATION_NUMBER : valeur entière comprise entre 0 et 255.

  • getInfoInt() est disponible dans l'éditeur d'expressions pour l'affichage numérique, l'affichage de chaîne, l'affichage de message, le sélecteur, et le commutateur.

  • Lorsqu'elle est utilisée dans un objet d'affichage de données, la valeur d'information ne sera pas mise à jour automatiquement. Vous devez actualiser l'écran (par exemple, en utilisant un changement d'écran) pour obtenir la valeur la plus récente.

  • Lorsque vous utilisez la valeur retournée, par exemple dans un objet d'affichage de chaîne, assurez-vous que la longueur d'affichage est configurée à 100 pour afficher tous les caractères. Pour les chaînes retournées plus longues que 00 caractères, seuls les 100 premiers caractères sont affichés.

  • getInfoInt() retourne toujours 0 lorsqu'elle est utilisée dans Web Gate.

  • L'ID du numéro de station est défini par les commutateurs rotatifs de la boîte de jonction. Pour plus d'informations sur les commutateurs rotatifs de la boîte de jonction, reportez-vous au manuel de la boîte de jonction qui est disponible sur le DVD de Vijeo Designer.

  • La somme de contrôle de l'application peut également être récupérée à l'aide d'une opération. Pour plus d'informations, reportez-vous à la section 9.5.13.2, Somme de contrôle.

  • Le numéro de station peut également être récupéré à l'aide d'une opération. Pour plus d'informations, reportez-vous à la section 9.5.13.1, Numéro de station.
