18.7.2.2 Opération Script

L'affichage de fenêtres popup est plus facile lorsque vous utilisez des commutateurs ou une animation sur appui. Dans certaines circonstances toutefois, vous voudrez peut-être qu'un autre événement déclenche l'affichage d'une fenêtre popup. C'est le cas des scripts.

Dans le script suivant, la fenêtre popup s'affiche lorsque la valeur d'une variable change. La variable change suite au déclenchement d'une alarme.

  • Vous pouvez utiliser des actions de déclenchement d'une alarme (reportez-vous à la section 16.3.4 Configuration des actions d'alarme) pour définir la fonctionnalité décrite dans cet exemple. Toutefois, avec ce type d'action, vous devez configurer la même opération pour chacune des variables. Ainsi, si vous avez 100 alarmes, vous devrez configurer 100 actions sur déclenchement. Cet exemple propose une configuration unique pouvant être utilisée pour un grand nombre d'alarmes.

18.7.2.3 Configuration des variables et de la fenêtre popup

Variables

Nom

Description

Erreur

Variable tableau du type BOOL comportant 11 éléments, utilisée pour effectuer le suivi des alarmes déclenchées.

Affectez des adresses de périphérique (00100 à 00110) à chaque élément du tableau.

Error[0] = 00100 ... Error[10] = 00110.

AlarmsActive

Variable BOOL interne.

Lorsque l'un des éléments de la variable Error est activé, AlarmsActive l'est aussi.

Cette variable joue le rôle de déclencheur pour l'affichage de la fenêtre popup.
