clearAll

Fonction

Le but de cette méthode de script est de retirer un message au complet du système.

La méthode de script clearAll permet d'effacer toutes les alarmes dans le résumé d'alarmes actuel, ou d'effacer tous les événements dans le résumé d'événements actuel. L'effacement retire tous les cas de ces alarmes à partir de toutes les listes d'alarmes (Actif, Historique, et Journal).

Les variables restent dans l'état d'alarme, mais les messages d'alarme associés n'apparaissent plus dans l'affichage du résumé d'alarmes. Même si les alarmes changent d'état, comme de Haut à Très haut, ces messages d'alarme n'apparaissent pas.

Ce comportement dû au forçage des alarmes de revenir à un état normal. Par la suite, un message d'alarme ne peut être généré à nouveau qu'après que l'alarme soit retombée à la normale. Les alarmes retombées à la normale ne peuvent pas être effacées par la méthode de script clearAll.

Utilisez un paramètre pour garantir que seul le type d'alarme spécifié est effacé.

  • Lorsque vous l'utilisez avec un groupe d'alarmes de diagnostic, cette méthode de script ne touche pas les alarmes dans le buffer de diagnostic de l'automate. Cette méthode de script n'interagit qu'avec l'affichage du résumé d'alarmes. Vous pouvez utilisez cette méthode sur tous les messages d'alarme dans les affichages d'alarme historiques et de journal, et sur les messages d'alarme acquittés dans les affichages d'alarme actifs.

Format

int NomRésuméAlarme.clearAll(int Msg_State, int Disp_State);

Paramètres

Paramètre

Description

int Msg_State

Spécifie le type d'alarmes effacé dans cette opération.

0: Permet d'effacer tous les messages d'alarme des états Actif, ACQ, et NONACQ dans le résumé d'alarmes. Les messages d'alarme de l'état Retombé ne sont pas effacés.

1: Permet d'effacer tous les messages d'alarme d'état Acquitté.

int Disp_State

Spécifie la plage des alarmes effacées dans cette opération.

0: Permet d'effacer tous les messages d'alarme de l'affichage du résumé d'alarme.

1: Permet de n'effacer que les messages d'alarme visibles de l'affichage du résumé d'alarme. En d'autres termes, les alarmes de la page courante. Pour les affichages de résumé d'alarme de journal et lorsque Msg_State = 0, retire également toute alarme active, acquittée, ou non acquittée associée avec les alarmes effacées.

  • Lorsque vous effacez une alarme, son état est modifié à Retombée à la normale (Retombée) et exécute des actions d'alarme associées.

Si le comportement de l'alarme = ACQ facultatif :

  • Effacer l'alarme exécute des actions configurées avec la condition Lorsque l'alarme revient à la normale.

  • Lorsque l'alarme est en un état actif, effacer l'alarme exécute également des actions configurées avec la transition Actif à retombée.

  • Lorsque l'alarme est en un état ACQ, effacer l'alarme exécute également des actions configurées avec la transition ACQ à retombée.

Si le comportement de l'alarme = ACQ obligatoire :

  • Effacer l'alarme exécute des actions configurées avec la condition Lorsque l'alarme revient à la normale.

  • Lorsque l'alarme est en un état actif, effacer l'alarme exécute également des actions configurées avec la transition Actif à ACQ et ACQ à Retombée.

  • Lorsque l'alarme est en un état ACQ, effacer l'alarme exécute également des actions configurées avec la transition ACQ à retombée.

  • Lorsque l'alarme est dans un état NONACQ, effacer l'alarme exécute également des actions configurées avec la transition NONACQ à retombée.

Valeur retournée

Retourne un entier correspondant au nombre d'alarmes effacées dans cette opération.

Voir également

clear, delete, deleteAll

  • Si vous ouvrez un projet créé sous la version 4.1 ou sous une version antérieure, toute utilisation de clearAll dans un script sera remplacée par deleteAll.
