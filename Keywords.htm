18.3.6 Mots clés

Utilisez les mots clés afin d'ajouter de la logique et de contrôler le flux des opérations dans les scripts Vijeo-Designer .

Vous ne pouvez pas utiliser les mots clés Vijeo-Designer et les mots clés Java comme noms d'objets ou de variables.

Mots clés du contrôle de flux

Mot clé

Description

break

Quitte la boucle actuelle ou l'instruction de cas.

do-while

Répète un ensemble d'instructions tant que l'expression conditionnelle est true. Les instructions sont au moins exécutées une fois.

for

Répète un ensemble d'instructions tant que l'expression conditionnelle est true.

Sténo pour une boucle, pour laquelle les variables sont initialisées avant que la boucle et les variables soient mises à jour à chaque itération de la boucle.

if-else

Contrôle le flux d'une exécution à l'aide d'une condition.

return

Quitte le script.

switch-case-default

Exécute un ensemble d'instructions de cas qui correspondent à la condition d'interruption définie. Exécute des instructions par défaut lorsque la condition d'interruption ne correspond pas à l'une des instructions de cas.

while

Répète un ensemble d'instructions tant que l'expression conditionnelle est true.

Lorsque vous utilisez des boucles (do-while, for, while), veillez à ne pas créer une boucle sans fin. Les boucles sans fin, impossibles à détecter lors de la validation, causent un blocage fatal de la machine cible étant donné que le script utilise toujours plus de mémoire.

Autres mots clés

Outre les mots clés de contrôle de flux, les scripts Vijeo-Designer prennent en charge les mots clés suivants.

Mot clé

Description

false

Valeur booléenne, utilisée couramment dans les comparaisons les affectations.

new

Crée des tableaux, des objets String ou StringBuffer.

null

N'indique aucune valeur, utilisé couramment dans les comparaisons les affectations.

true

Valeur booléenne, utilisée couramment dans les comparaisons les affectations.
