18.8 Utilisation du pilote de script

Le pilote de script est un pilote de communication série générique qui vous permet de communiquer avec de l'équipement série simple. Le pilote de script avec les méthodes read et write de Vijeo Designer, vous permettent d'envoyer et de recevoir des flux d'octets de données par le port série.

Pour plus d'informations sur la structure et les paramètres des méthodes read/write, reportez-vous à la section 18.8.2, Méthodes pour les pilotes de script.

Pourquoi utiliser un pilote de script ?

Le pilote de script est utilisé pour les communications série, telles que la communication avec un hyperterminal Windows, l'envoi de messages à une imprimante, ou l'envoi de messages à un équipement d'affichage de texte. Sur ces équipements distants, vous pouvez afficher d'importantes informations sur le statut (nombre actuel d'alarmes, de barres d'indicateurs créées, inventaire restant, etc.).

Schneider Electric vous recommande d'utiliser le pilote de script uniquement pour les communications simples.

  • Avertissement : Si vous exécutez le script très fréquemment dans le but de lire et d'écrire un grand nombre de caractères à l'aide du pilote de script, il se peut que d'autres composants de l'application Runtime soient ralentis.
