write

Fonction

Envoie la chaîne définie comme flux d'octets au port de communication spécifié.

Vous pouvez utiliser les méthodes read/write du pilote de script pour réaliser les communications de base entre la machine cible et un équipement. Cet équipement peut être une imprimante, un équipement ou une autre machine cible.

Format

boolean NomPiloteScript.write(

boolean clearBuffer,

int numOfBytes,

byte[] data );

Paramètres

Paramètre

Description

boolean clearBuffer

A Vrai, efface toute donnée du buffer de réception de la machine cible (buffer du pilote de script) avant la transmission.

  • Utilisez ce paramètre lorsque vous attendez une réponse de l'équipement série. Configurez ce paramètre à Vrai afin de retirer toutes les données stockées dans le buffer de réception, qui donc facilite l'analyse des données reçues.

  • Si l'équipement série connecté poursuit l'envoi et la réception des données, ce paramètre devrait être false les données dans le buffer du pilote de script ne sont pas effacées. Il existe peut-être des données qui devraient toujours être lues qui ont été envoyées par l'équipement connecté.

int numOfBytes

Définit le nombre d'octets dans la transmission de données. Vous pouvez écrire jusqu'à 1024 octets de données dans une transmission unique.

  • Essayez de configurer les applications pour que vous ne deviez pas écrire des montants importants de données. Toutes les opérations sur la machine cible sont bloquées jusqu'à ce que l'opération d'écriture soit terminée.

  • Lorsque vous écrivez 1024 octets avec une vitesse de transmission de 19200 bps, la machine cible arrête de s'exécuter pour 2 à 3 secondes.

byte[] data

Données envoyées au port de communication telles que définies dans le pilote de script.

Valeur retournée

Retourne true lorsque les données sont écrites avec succès dans le buffer du pilote. Sinon, renvoie la valeur false.

Commentaires

Une valeur vraie ne garantie pas que les données ont été écrites avec succès dans le périphérique. Une valeur vraie indique seulement que les données ont été écrites avec succès dans le buffer du pilote.

La méthode write bloque l'exécution de toutes autres opérations lors de l'envoi des données via le port série.

Pour utiliser les méthodes read/write de pilote de script, vous devez tout d'abord configurer un pilote de script. Reportez-vous à la section 18.8.3, Configuration du pilote de script.
